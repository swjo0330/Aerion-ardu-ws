# 5단계 검증 게이트 + 리뷰어 체크리스트 — 설계 정본

> 작성 2026-07-10 · 상태: 사용자 승인(2026-07-10) · 스코프: **ardu_ws 하네스 한정**
> 목적: "범위 잡기 → 근거 수집 → 문제 풀기 → 검증 → 보고" 5단계를 **품질·신뢰성 게이트**로 명문화 — 실력의 절반은 두뇌가 아니라 꼼꼼한 일하는 방식에서 나온다는 취지(사용자 제시 이미지). 발동·기록은 기존 하네스(FABLE5·CLAUDE §1.5·notes) 재사용, 병렬 규범 신설 금지.

## 1. 결정 요약

| 항목 | 결정 | 근거 |
|---|---|---|
| 성능향상 정의 | **품질·신뢰성 게이트 강화** (속도 아님) | 사용자 Q1 확정 |
| 리뷰어 형태 | **자가체크(상시) + 적대검증 서브에이전트(대형·비가역)** | 사용자 Q2 확정 |
| 배치 | **FABLE5.md 확장 + Docs/reviewer-checklist.md 신설**, 이 프로젝트 한정 | 사용자 Q3 확정 |
| 핵심 설계 원칙 | 5단계를 새 구조로 세우지 않고 **기존 하네스에 매핑** — ①③은 이미 존재(사전점검·최소diff), **②④⑤만 게이트 신설** | 이중 규범=사문화 위험 |

## 2. 5단계 ↔ 하네스 매핑 (정본은 FABLE5.md §5단계 검증 게이트)

- ① 범위 = 사전점검 블록 그 자체 (기존) + "모호점 잔존 시 착수 금지" 명문화
- ② 근거 = **신설**: 핵심 주장마다 근거유형 태깅 `[file:line | 실측로그 | 사용자확정 | 미확인]` — 미확인 상태로 ③ 진입 시 notes에 [가정] 선기록
- ③ 풀기 = 기존 (최소 diff·1수정 1재검증·가역성)
- ④ 검증 = **신설**: 완료 선언 전 검증 명령 실제 실행+출력 인용 의무. ardu_ws 표준 증거 카탈로그는 reviewer-checklist.md에 정본
- ⑤ 보고 = **신설**: 결과+한계 병기, 증거 ID 필수, 미검증="미확인" 명시

발동 차등(과잉 방지): 대형=5게이트+말미 `✅ 게이트` 1줄 / 중형=②④⑤ 축약 / 소형=생략 / 반복 정형(시뮬 재시작 등)=④만.

## 3. 리뷰어 체크리스트 (정본은 Docs/reviewer-checklist.md)

- L1 자가체크: 5단계×3~4항목, 대형 작업 종료 시 메인이 수행 → 말미 1줄 선언. 실패 항목 있으면 완료 선언 금지.
- L2 적대검증 서브: 대형·비가역·크로스머신 규격 변경 시. `harness_template/prompts/adversarial-review.md` 패턴 + ardu_ws 특화 프롬프트 템플릿(표준 증거 카탈로그 연동). 판정 5분류.

## 4. 발동 배선 (사문화 방지)

CLAUDE.md §1.5(완료 선언 전 자가체크 의무 1줄) → FABLE5.md(§게이트 절) → Docs/reviewer-checklist.md(체크 항목·증거 카탈로그·서브 프롬프트) → session-restart-protocol 종료 자기감사(게이트 무선언 완료 → [이탈]).

## 5. 검증 (설계 자체)

- 배선 체인 링크 실재 + placeholder 0 (grep)
- 리트로 테스트: 이번 세션 실패 2건 — (a) navsat 이중계산(home+gz 동시 오프셋 → 60m): ② 게이트의 "수치 주장 실측 태깅"에서 검출됐어야 → 체크 항목 "물리량 합성 경로에 이중 기여 없는가" 포함 (b) lock_step RTF 스톨: ④ 게이트의 "기동 후 sim time 진행 실측"에서 검출 → 표준 증거 카탈로그에 RTF 확인 포함. 두 건 모두 체크리스트가 커버함을 확인 후 완료.
