# 리뷰어 체크리스트 — 5단계 게이트 (ardu_ws)

> 발동: [CLAUDE.md](../CLAUDE.md) §1.5 (대형 작업 완료 선언 전 의무) · 게이트 정의: [FABLE5.md](../FABLE5.md) §5단계 검증 게이트 · 설계 근거: [specs/2026-07-10-five-stage-gate-design.md](specs/2026-07-10-five-stage-gate-design.md)
> **L1 자가체크**는 메인이 매 대형 작업 종료 시 스스로 수행. **L2 적대검증 서브**는 대형·비가역·크로스머신 규격 변경 시 추가.

---

## L1. 자가체크 (대형 작업 종료 시 — 전 항목 통과해야 완료 선언 가능)

### ① 범위
- [ ] 요구를 1문장으로 재정의했고, 사용자와 어긋날 소지가 있는 해석은 착수 전에 물었다
- [ ] 이번 작업의 경계(포함/제외)가 분명하다 — 범위 밖 발견은 수정하지 않고 기록만 했다
- [ ] 착수 시점의 Blind Spot(위험 가정 1~3개)을 명시했다

### ② 근거
- [ ] 핵심 주장(수치·동작·경로)마다 근거유형이 붙어 있다: `file:line` / 실측로그 / 사용자확정 / **미확인**
- [ ] "미확인"을 사실처럼 단정한 곳이 없다 — 미확인 상태로 진행한 것은 notes에 `[가정]`으로 선기록했다
- [ ] 기억(메모리)·문서 인용은 **현재 실물로 재검증**했다 (기억=시점 관찰)
- [ ] **물리량·수치의 합성 경로에 이중 기여가 없는가** (예: navsat = home + gz_pos — 양쪽에 같은 오프셋을 주면 이중계산. 2026-07-09 실증)

### ③ 풀기
- [ ] 최소 수직 슬라이스로 갔다 (큰 것 한 번에 X) — 1수정 1재검증
- [ ] 비가역·불가침 규약(CLAUDE §1)·크로스머신 인터페이스 영향은 기록 후 사용자에게 물었다
- [ ] src/install 이중 트리 대상이면 **양쪽 동시** 수정 또는 재빌드했다

### ④ 검증
- [ ] 검증 명령을 **실제 실행**했고 출력을 인용할 수 있다 — "계획만 쓰고 생략"이 아니다
- [ ] 아래 표준 증거 카탈로그에서 해당 항목을 골라 실행했다
- [ ] 수정 후 회귀 게이트(단일 모드 무결 등)를 통과했다
- [ ] 관찰 도구 자체를 의심했다 (예: `ros2 topic list`의 /ap 열거 플레이크 → echo로 판정)

### ⑤ 보고
- [ ] 결과와 한계(hedge)를 **같은 자리에서** 병기했다 — 검증 안 된 것은 "미확인" 명시
- [ ] 완료 주장마다 증거 ID(명령 출력·로그 라인·커밋 해시)가 붙어 있다
- [ ] 실패·스킵도 그대로 보고했다 (과장·생략 없음)
- [ ] 기록 동기화: notes 5태그 / EXPERIMENTS 트랙 / 메모리(해당 시)

**통과 시**: 응답 말미 `✅ 게이트: ①~⑤ 통과 (증거: <핵심 1~2개>)` 1줄. **한 항목이라도 실패 시 완료 선언 금지** — 잔여로 보고.

## 표준 증거 카탈로그 (④에서 선택 실행 — ardu_ws 정본)

| 대상 | 명령 | 통과 기준 |
|---|---|---|
| 단일 모드 기동 | `bash start_sim.sh` 후 로그 | `DDS: Initialization passed` + `EKF3 active` + 바인딩 에러 0 |
| 멀티 모드 기동 | `bash start_multi_sim.sh 3` 후 logs/multi_i*.log | 위 시그니처 ×3 |
| **sim 물리 진행** | `/clock` 2회 샘플(≥3s 간격) | sim time 전진 (RTF 스톨 검출 — lock_step 사례) |
| 도메인 분리 | `ROS_DOMAIN_ID=N ros2 topic list --no-daemon` ×3 | 자기 기체 토픽만, 교차 오염 0 |
| 실데이터 (list 플레이크 우회) | `ROS_DOMAIN_ID=N ros2 topic echo /ap/time --once` | 실데이터 수신 |
| GPS 이격 | 도메인별 `/ap/navsat` lat/lon → 쌍간 거리 계산 | 규격서 §7 값과 일치 |
| 스폰 대형 | `gz topic -e -t /world/map/dynamic_pose/info -n 1` | 설계 오프셋과 일치 |
| 문서 placeholder | `grep -rEn '\{\{' <대상>` | 0줄 |
| 하네스 링크 체인 | 참조 파일 `ls` + 앵커 대조 | 전부 실재 |
| 이중 트리 정합 | `diff src/... install/...` 또는 재빌드 | 불일치 0 |
| 크로스머신 규격 push | `gh api repos/.../contents/...` | 원격 반영 확인 |

## L2. 적대검증 서브에이전트 (대형·비가역·크로스머신 규격 변경 시)

패턴 정본: [harness_template/prompts/adversarial-review.md](../harness_template/prompts/adversarial-review.md). ardu_ws 특화 프롬프트 템플릿:

```
다음 산출물을 적대적으로 검증하라: <산출물 경로/설명>
1) 산출물의 핵심 주장을 열거하고, 각각을 REFUTE 시도하라 — 근거 없는 지적 금지(지적마다 file:line/실측 근거 필수).
2) 검증 명령은 Docs/reviewer-checklist.md의 표준 증거 카탈로그에서 골라 실제 실행하고 출력을 인용하라.
3) 특히: 수치 합성 경로 이중 기여 / src·install 이중 트리 불일치 / "미확인"의 사실 단정 /
   검증 명령 미실행 완료 선언 / 크로스머신 인터페이스(규격서)와의 모순을 노려라.
4) 현 상태가 강점일 가능성도 고려하고, 기각 시 기각 근거를 남겨라.
5) 판정 5분류로 보고: 수용 / 부분수용 / 기각 / 보류 / 범위밖. 확인 못 한 것은 "미확인"으로 표기.
```

- 서브 판정을 메인이 재검증한 뒤에만 사용자 보고에 반영 (서브 결과 맹신 금지)
- 발동 판단 기준: ①되돌리기 어려운가 ②저쪽(.29)이 소비하는 규격인가 ③다파일·다트리 변경인가 — 하나라도 예면 L2

## 리트로 검증 (이 체크리스트의 근거 — 2026-07-09 실사례 2건)

| 실패 사례 | 게이트가 잡는 지점 |
|---|---|
| navsat 이중계산 (home+gz 동시 오프셋 → 의도 30m가 60m) | ②-4 "합성 경로 이중 기여" + ④ GPS 이격 실측 |
| lock_step=1 RTF 0.2% 스톨 (기동은 됐는데 sim time 정지) | ④ "sim 물리 진행" — 시그니처만 보고 통과 선언하면 놓침 |
