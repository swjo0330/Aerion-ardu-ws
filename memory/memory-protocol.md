| **② 파일 메모리 — 진실원천** | `~/.claude/projects/-Users-swjo-yonsei-ai-aerion/memory/*.md` + `MEMORY.md` 인덱스 | 매 세션 (인덱스 주입) | **유저 선호(feedback_)·진행 상태(project_)·환경/도구(reference_)·신원(user_)** — 사람이 읽는 정본 | 1파일 = 1사실 |

**라우팅 1원칙 — "수명과 형태로 고른다"**:
- *항시 강제되어야 하는 한 줄 규칙* → ① (가장 비싼 자리, 정말 불변일 때만 — 초안 제시 후 유저 승인)
- *나의 행동을 바꾸는 선호/정체성, 세션 간 상태, 재인용할 기술 사실·결정* → ② (진실원천)
- *서술이 필요한 긴 기록(스펙·실험·셋업)* → ④, 그리고 ②에 1줄 포인터
- ③ memoir는 라우팅 대상이 아니다 — 자동 캡처가 알아서 담는 **보조 채널**. ②와의 약간의 중복 적재는 용인 (실해 없음)
- ⑤는 기록처가 아니라 **②의 동기화 대상** — 아래 §0-2 참조

**중복 방지**: 동일 사실을 여러 저장소에 전문 복제하지 않는다 — **긴 서사는 ④, 짧은 회상 훅은 ②(인덱스 1줄)**. 이미 있으면 신규 생성 대신 갱신.

### 확정 운영 결정 (2026-06-16 사용자 확정 — **재제안 금지**)

- memoir 플러그인은 **끄지 않는다** — 자동캡처+회상 병행 ON 유지
- Stop 훅 / `MEMOIR_NO_CAPTURE` / **저장소 일원화 — 추가하지 않는다** (복잡화 배제)

### ② 파일명 컨벤션 (기존 접두사 유지)

- `feedback_<증상or우회책>.md` — 비자명한 에러의 원인·해결법·우회책·팁 (예: `feedback_dds_large_msg.md`, `feedback_ros2_daemon_trap.md`)
- `project_<이름>.md` — 프로젝트 상태·구조·진행 현황 (예: `project_ardu_ws.md`, `project_distance_sensors.md`)
- `reference_<카테고리>.md` — 환경·도구·정책, 변경 잦지 않음 (예: `reference_claude_code_env.md`)
- `user_<항목>.md` — 사용자 신원·선호·제약

크기 100~500 단어(핵심만). 기존 파일 먼저 검색 → 같은 주제면 새 파일 금지, 갱신(dedup). frontmatter(`name`/`description`/`metadata`) 유지.

### 기록/SKIP 판정 (DETECT)

재사용성 ∧ 비자명성 ∧ 지속성 **모두** 충족 시만 기록. 판정 3문:

1. "이 정보가 없었으면 다음에 같은 문제에서 1시간 낭비했을 텐데?" → **YES → 기록**
2. "git log / 소스코드 읽으면 알 수 있는 건가?" → **YES → SKIP**
3. "이 세션 하나에만 필요한 스크래치/일회성 디버깅인가?" → **YES → SKIP**

**애매하면: 저장** (미저장 비용 > 과저장 비용).

## 0-1. 자율성 경계 + 투명성 계약

자율성은 "되돌릴 수 있고 내부에만 영향"인 쓰기에만 부여한다.

| 행위 | 자율 등급 | 통지 방식 |
|------|----------|----------|
| ②④⑤ 기록·갱신·동기화 | **말없이 자동** (가역적 내부 지식) | 응답 말미 `🧠 기억: <slug> — <훅>` 1줄 |
| ① CLAUDE.md 불변 규칙 신설/변경 | **초안만 제시** — 유저가 반영 | "CLAUDE.md §N 추가 제안 — 붙여넣을 텍스트입니다" |
| ⑤의 git **commit** | **자율 아님** (CLAUDE.md §1: 커밋은 사용자 요청 시에만) | 파일 복사까지만 자동, 커밋은 제안 |
| 코드/설정 수정, 외부 전송 | **자율 아님** (유저 승인) | 해당 안 됨 |

**유저 거부권(veto)**: "그거 지워 / 기억하지 마 / 틀렸어" → 즉시 삭제·정정. 자동 기록은 항상 가역적이어야 한다.

## 0-2. ⑤ git 사본 동기화 정책 (`.claude/RESTORE.md` 정본)

- ②(글로벌 파일 메모리)가 **원본**, ⑤ `.claude/memory/`는 git tracking용 **이식 스냅샷**
- **의미 변화 발생 시에만 ②→⑤ 복사** — 워크플로우/구조/트러블슈팅처럼 재발 시 다시 알아내야 하는 정보만
- **IP 등 매일 변동하는 휘발 정보는 커밋 금지** (커밋 가치 없음 + 스냅샷 오독 위험)
- commit 자체는 사용자 요청/승인 하에 (§0-1)
- 시점: 세션 종료 배치 패스 (session-restart-protocol.md 체크리스트)

## 실행 원칙 6종

1. **자연 경계에서 기록**: 트리거 ①②④는 마일스톤·방향 전환·세션 종료에 일괄. 즉시는 ③⑤⑥만
2. **중복 회피**: 신규 생성 전 `MEMORY.md` 인덱스 확인 → 기존 갱신 우선 (memoir 키는 확인 대상 아님 — 자동 채널)
3. **틀린 기억 삭제**: 정정된 사실은 옛 기록을 직접 수정·삭제 (취소선 보존 금지 — 누적 오염 방지)
4. **인덱스 동기화**: 메모리 파일 추가/변경 시 `MEMORY.md` 1줄 포인터 갱신 (순수 인덱스, <17KB)
5. **확신 없으면 묻지 말고 기록 후 통지**: 사소한 판단은 수행 + 1줄 언급, 영향 큰 것만 확인 요청
6. **조용히**: 기록 행위를 장황히 보고하지 않음 — 말미 1줄이면 충분

## 세션 종료/장기 정리 패스 (누적 부패 방지)

- **트리거**: 긴 세션 마무리, compaction 직후, 유저 "정리해"
- **동작**: ①같은 주제 파편 병합 ②거짓이 된 기록 삭제 ③인덱스↔실파일 정합 점검(끊긴 포인터) ④의미 변화분 ②→⑤ 동기화 (§0-2, 휘발 정보 제외)
- **원칙**: 정리도 가역적·말없이 자동. 단 삭제는 `🧹 정리: <slug> 병합/삭제` 1줄 통지
