# 발표 덱 규칙 카드

> 실운영 프로젝트의 덱 생성 스크립트에서 실제 사용 중인 값·구조만 추출. 함께 사용: [prompts/deck-discipline.md](../prompts/deck-discipline.md), 검수는 [prompts/visual-qa.md](../prompts/visual-qa.md).

## P1. 색 팔레트 (hex 고정)

본색: NAVY `#1F3864` · TEAL `#0E7C7B` · GREEN `#2C7A51` · ORANGE `#C05621` · GRAY `#4A4A4A` · LGRAY `#8A8A8A` · DARK `#2B2B2B`(로그 인용 배경).
파스텔 배경: NAVY_BG `#EFF2F9` · TEAL_BG `#EEF6F6` · GREEN_BG `#EFF7F1` · ORANGE_BG `#FDF3EA` · GRAY_BG `#F4F5F7` · 테두리 `#D9DEE5`.

**색 의미론 고정**: navy=구조·주축, green=긍정 결과·실험군, orange=주의·남은 것·정직 표기, gray=대조군·중립, teal=보조 축.

## P2. 폰트·판형

16:9 (13.333×7.5in), 한글 고딕 단일 폰트 <!-- 예: Apple SD Gothic Neo -->. 슬라이드 제목 25pt bold NAVY / 소제목 12.5~13pt bold / 본문 10.3~11.5pt GRAY / 각주 9pt LGRAY. 카드=둥근사각형, 그림자 제거, 테두리 없음 또는 1pt.

## P3. "왜/결과" 2줄 헤더 (전 슬라이드 필수)

제목 아래 2줄: `왜 {{이 실험/그림이 필요한 이유 1문장}}`(라벨 NAVY bold) + `결과 {{수치 포함 완결 결론 1문장}}`(라벨 GREEN bold). 결과 줄은 반드시 구체 수치를 담은 완결문 — "잘 됐다" 금지.

## P4. 해석박스 3단 규율

초록 파스텔 카드, 헤더 "이 그림이 말하는 것"(GREEN bold). 불릿 순서 고정: ①`읽는 법: {{축·칸·색이 무엇인지}}` → ②`{{그림에서 보이는 것}}` → ③`{{그것이 의미하는 것 + 한계}}`. **그림에 없는 사실 금지** — 그림 밖 정보는 "오른쪽 상자 참조"로 위임.

## P5. 우측 사이드 카드 (흰 배경+테두리)

소제목 섹션 구조 고정: "읽는 법" / "핵심 결과" / "정직 표기"(또는 "표기 규율" — ORANGE 헤더). 한계·단일시행·해상도 하한은 반드시 "정직 표기" 소절로 결과 옆에 병기.

## P6. run ID 각주 (수치 슬라이드 필수)

하단 9pt LGRAY: `run: {{실행 ID}} ({{n회}}) — {{산출물 성격}}`. 수치를 보인 슬라이드에 실행 ID 각주 누락 금지. 같은 데이터 재사용 시 "run 각주는 앞 장과 동일" 명기.

## P7. 파트 태그 배지

좌상단 소형 컬러 배지 `Part n — {{축 이름}}`, 파트당 색 고정(축 1=NAVY, 축 2=GREEN 식).

## P8. 표지 공식

큰 제목 + 부제 1줄 → 핵심 메시지 3카드(각 축 색, 헤드라인 16pt bold + 본문 2줄 11.5pt) → 하단 GRAY_BG 스트립 `수치 규율: {{인용 가능 조건 + hedge 상태}}`.

## P9. 청중 언어 이중화

본문은 평이한 한글 명칭, 내부 코드명은 `(내부 명칭 {{코드명}})` 소자(9.5pt LGRAY)로만. <!-- 예: "결합 검색 + 결과 기록 … 내부 명칭 (M-out)" -->

## P10. 오해 선제 차단 카드

직관에 반하는 수치는 전용 카드로 선제 해명: `{{수치}}는 느린/나쁜 게 아니다` 헤드 + `지배 요인 = {{설계값}}` 본문. <!-- 예: "승격 ~10초는 느린 게 아니다 — 상태 신선도 계약 10초" -->

## P11. 진행판 슬라이드 = 3열 매트릭스

행=축, 열 헤더 고정: "완료 (이번 주까지)"(GREEN) / "핵심 수치"(NAVY) / "남은 것"(ORANGE).

## P12. 제작 파이프라인 분기 (2026-07-09 실증 갱신)

| 용도 | 파이프라인 |
|---|---|
| **시각 품질 우선** (발표·데모·제출용 — 기본 권장) | **HTML/CSS 직접 코딩 → 헤드리스 Chrome 2배 스케일(3840×2160) 스크린샷 → JPEG(q76) → img2pdf로 PDF** (+필요 시 이미지 기반 pptx). 근거: CSS 레이아웃 엔진이 배치를 계산 → python-pptx의 겹침·패딩·정렬류 QA 결함이 구조적으로 회피됨. 타이포그래피 자유도(웹폰트·자간) 우수 |
| 수신자가 **편집해야 하는** 덱 | python-pptx 조립 — 단 렌더→육안→수정 QA 루프(P1~P11) 필수 |
| 데이터 차트 | 어느 쪽이든 matplotlib(현지어 폰트) 재생성 후 삽입 |
