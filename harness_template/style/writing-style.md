# 문서 글쓰기 규칙 카드

> 실운영 프로젝트의 기록/실험 문서에서 실제 사용 중인 패턴만 추출. 규칙은 측정 가능하게(위반 판정이 기계적으로 되게) 기술.

## 1. 문서 스타일

**D1. 파일명·제목** — 파일명 `YYYY-MM-DD-{{키워드}}.md`. H1 = `# YYYY-MM-DD {{장소/컨텍스트}} — {{주제}} ({{판형 태그}})` <!-- 예: 판형 태그 = "간결판" -->

**D2. 첫 블록 = blockquote 메타 헤더 (본문 시작 전 필수)**
```
> 수치 정본 = [{{정본 문서명}}]({{경로}}) ({{이 문서와 정본의 역할 구분 1구}}).
> 환경: {{핵심 환경 1줄}} · {{알려진 교란요인 + 사용자 결정 + 대응책}}
```
역할 선언 문장 포함 — "이 문서는 지도·현황판, 수치 인용은 반드시 정본에서" 식으로 이 문서가 정본이 **아닌** 것을 명시. 현황판 문서는 갱신 책임("상태가 바뀐 세션이 §N을 갱신")까지 헤더에 둔다.

**D3. 정본 단일화 (Single Source of Truth)** — 사실·수치마다 정본 1곳, 나머지 문서는 링크 포인터만. 수치를 재인용하는 표·그림에는 각주로 정본 위치 재명시. <!-- 예: "(런 ID·수치 정본 = registry)" -->

**D4. 의미 서술 단락 패턴 (실험/작업 항목)**
```
**{{조건 이름}} ({{내부 코드명}} — "{{평이한 질문형 별칭}}")**
{{무엇을 했나 평문 1~2문장}}. **효과: {{관찰된 결과 — 수치 포함 볼드 1문장}}** — {{근거 수치}}. {{한계 단서}}.
```
결과 문장은 반드시 볼드, 한계 단서는 결과와 **같은 단락**에 붙인다(분리 금지). <!-- 예: 한계 단서 = "단 아직 4쌍이라 재현 시그널 단계(6쌍에서 통계 확정)" -->

**D5. 표 중심 + 표 하단 이탤릭 각주** — 지표↔결과 매칭·함정↔조치·항목 명세는 산문 대신 표. 표 열에 반드시 "의미(무엇을 재나)" 열과 "상태" 열 포함. 표 바로 아래 `*({{주의사항 — 비교금지 조건·정본 위치}})*` 이탤릭 각주. <!-- 예: "(A수치와 B수치는 모델이 달라 서로 비교 금지)" -->

**D6. 그림 캡션 규율** — 이미지 바로 아래 `*(그림: {{그림이 직접 보여주는 관찰 사실 1문장}})*`. 그림에 없는 해석·주장 금지.

**D7. 이모지 = 상태 배지 한정** — 허용: ✅완료 ⬜예정 🟡진행 🔴긴급 ★확정(등급). 장식·감정 이모지 0개.

**D8. 한/영 병기** — 한글 본문 우선. 기술 용어 첫 등장 시 괄호 영문 1회 <!-- 예: "기억 분해 실험(memory ablation)" -->, 코드 식별자·로그 원문은 백틱. 로그 원문 인용은 증거로서 그대로 백틱 인용.

**D9. 수치 정직 규율 (hedge 문법)** — n 미달 결과 = "재현 시그널(검정 전)"·"descriptive" 명기. 산출 불가 값 = null 정직 표기(0으로 채우지 않음). 무효 데이터도 `_INVALID` 라벨로 보존 사실 기록. 단정형은 측정 확정에만, 미측정 주장은 가설형("~하는지 검정").

**D10. 섹션 구조** — 번호 섹션(중간 삽입은 §1.5식 소수점) → 마지막 섹션 = "다음": ①②③ 원형숫자 인라인 리스트 1단락.

## 2. 기록(원장) 스타일 — 미결/결정 추적 파일

**R1. 1줄 원장 포맷**
```
- [유형] YYYY-MM-DD | {{대상}} | {{내용 1~2줄}} | ({{근거/포인터 — 파일:라인·런 ID·"사용자 확정"}})
```
최신이 위. 근거 필드는 생략 불가.

**R2. [유형] 태그 5종 고정**
- `[빈칸]` 구현 중 드러난 미해결 TBD·보류
- `[가정]` 검증 안 된 전제 — **근거 + 위험도(낮/중/높) 명기**, 실측 확정 시 닫음
- `[결정]` 선택과 근거 — 가역적 소결정=기록 후 진행, 비가역·아키텍처 영향=기록 후 사용자 질문
- `[이탈]` 정본 설계와 다르게 구현한 것 — **이유·위험도·가역 여부** 명기
- `[테스트]` 게이트·실측 결과 요지 — 수치 정본은 별도 정본 문서, 여기는 요지만

**R3. 아카이브 규율** — 해결·확정 항목은 하단 `## 아카이브`로 `- [닫힘] ~날짜 {{요지}} → {{정본화된 위치}}` 1줄 이동. 원장은 활성 항목만 유지.

**R4. 파일 자기 기술 헤더** — 파일 최상단 blockquote로 **역할**("나중에 '왜 이렇게 돼 있지?'의 답이 여기 남는다") · **형식**(R1 포맷 명시) · **유형 정의** · **운영**(아카이브 규칙 + 서사 정본화는 별도 채널 담당)을 자기 기술 — 규칙이 파일 안에 있어 로드만으로 준수 가능.

**R5. 사용자 결정 귀속** — 사용자가 정한 것은 반드시 `(사용자 확정)`/`(사용자 지시)`로 귀속 표기 — 에이전트 판단과 구분.

## 3. 횡단 규칙 (문서·덱·도식·원장 공통)

**X1. 정본-포인터 그물** — 수치는 정본 1곳 → 문서·덱·도식은 전부 포인터+캡션.
**X2. 결과 옆 한계 병기** — 결과 문장과 hedge(검정 전·단일 시행·비교 금지)를 같은 시각 단위(단락/카드/캡션)에 붙임. 분리 배치 금지.
**X3. 증거 ID 추적성** — 모든 수치 뒤에 런 ID / 파일:라인 / 로그 원문 중 하나.
**X4. 이중 명명** — 내부 코드명 ↔ 평이한 질문형 별칭을 항상 쌍으로. 문서는 코드명 우선+별칭 괄호, 덱은 별칭 우선+코드명 소자.
