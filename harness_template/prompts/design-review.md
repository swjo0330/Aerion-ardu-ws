# ⑪ 설계 검토·계획 수립 ("설계해서 보고해봐")

**언제 쓰나**: 새 기능·실험·구조의 설계를 잡아 사용자 결정을 받을 때. 구현 전 단계 — 산출물은 "결정 가능한 설계 보고".

## 프롬프트

```
{{대상}}의 설계를 수립해 보고하라.
1. 제약·정본 확인: 기존 설계 문서·불가침 규칙·과거 결정(기억 회상)을 먼저 — 이미 결정된 것을 재설계하지 마라
2. 대안 2개 이상을 표로 비교 (기준: 목적 적합성·비용·리스크·기존 자산 재사용)
3. 권고 1개 + 그 권고에 대한 반론(스스로 공격)과 방어 — 반론이 못 막히면 권고를 바꿔라
4. 구현 계획: 단계·각 단계의 게이트(무엇으로 검증)·리스크와 완화
5. 사용자 결정 포인트를 분리 명시: "여기는 내(에이전트) 판단으로 진행 가능 / 여기는 승인 필요(비가역·아키텍처·불가침 인접)"
6. 채택 시 설계 정본 문서로 기록({{specs 경로}}, 날짜-키워드 네이밍)
```

**실증**: AirSim 임무 스모크 파이프라인(S0~S6 단계+함정 체크리스트 → 즉시 구현 승인), 기억 분해 본런 설계(매트릭스+선결 3건+모델 분기 → 덱 슬라이드화), Qdrant 코호트 통제(스냅샷→조건런→복원 전략).
