# ⑤ 도식 제작 (블럭 다이어그램 + 해석 캡션)

**언제 쓰나**: 아키텍처·실험 구성·파이프라인을 그림 1장으로 설명해야 할 때. (상세 스타일 값은 [`../style/diagram-style.md`](../style/diagram-style.md) 참조)

## 프롬프트

```
{{대상 시스템/실험}}의 블럭 다이어그램을 제작하라.
- 구성: 블럭 = 컴포넌트, 화살표 = 데이터/제어 흐름(방향·채널 종류 라벨),
  경계선 = 실행 환경 구분 <!-- 예: 시뮬/실물, 로컬/클라우드 -->
- 편집 원본(SVG 등)과 렌더본(PNG/PDF)을 함께 산출 — 수정은 원본 편집→재렌더
- 함정: SVG font-size 속성은 CSS 클래스에 밀린다 → style 속성으로 지정
- 각 도식에 해석 캡션 필수: 무엇을 보여주는 그림인지 + 읽는 순서 1~2문장
- 병렬 생성 도구 사용 시 산출물을 작업 폴더로 즉시 복사(생성 캐시에 방치 금지),
  백그라운드 실행은 stdin 차단 필요 <!-- 예: codex exec ... </dev/null -->
- 완성본은 시각 QA(→ visual-qa.md) 통과 후 정본 문서에 포인터 등록
```

**실증**: 실험 구성도 5장+갤러리 7장을 병렬 도구로 생성(`</dev/null` stdin 함정·생성 캐시 회수 함정 실기록), 논문 그림 SVG 원본→Chrome headless 렌더 규약, SVG font-size CSS 우선순위 함정.
