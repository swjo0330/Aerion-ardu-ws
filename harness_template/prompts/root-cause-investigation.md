# ⑧ 원인 조사 (재현→경쟁가설→증거)

**언제 쓰나**: 버그·이상 동작·원인 불명 실패를 만났을 때, 첫 인상으로 고치기 전에.

## 프롬프트

```
{{증상}}의 근본원인을 조사하라. 규율:
1. 재현 먼저 — 결정론적 재현 조건을 확보(실측/로그)한 뒤에만 가설로 진행
2. 경쟁 가설 3개 이상을 세우고, 가설별 지지/반박 증거를 로그·실측에서 수집
3. 원인은 전체 인과 연쇄로 서술 (트리거 → 중간 메커니즘 → 관측 증상)
   <!-- 예: 오탐 → 승격 규칙 → 개입 발동, 의 3단 연쇄 -->
4. 수정 전/후 동작을 같은 재현 조건으로 비교 검증
5. 기각한 가설과 기각 근거도 보고 (오진 교훈 포함)
6. 확정된 함정은 즉시 함정 목록/기억에 기록
```

**실증**: 불필요 재계획 원인 = 에이전트 분석+실측으로 3단 인과 연쇄 확정(오탐→검색 승격→감독 강제 개입), 임베딩 데드락(2중 모델 로드, 수정 후 227s→22s 전후 검증), 연결 레이스 버그·`pgrep -f` 오진 교훈 기록.
