# ⑨ 프로젝트 심층분석·총검토 (갭 분석 → 단계 구상)

**언제 쓰나**: "지금까지 한 것 총검토해서 다음 준비해" — 트랙/프로젝트 전체를 조망하고 남은 일을 단계화할 때. 세션 재시작 회상(⑥)보다 깊고, 산출물이 "실행 가능한 단계 플랜"일 때.

## 프롬프트

```
{{트랙/프로젝트}}를 총검토하고 다음 단계를 구상하라.
1. 정본 지도 확인: 대시보드({{EXPERIMENTS.md 등}})·수치 정본({{registry}})·설계 정본을 실제로 읽어라 — 기억만으로 쓰지 마라
2. 현황 3분류: ✅확보(등록 완료·재현됨) / 🟡시그널(나왔으나 검정·확정 전) / ⬜미착수 — 각 항목에 근거(문서·run ID) 병기
3. 갭 판정: 목표({{논문/마일스톤 요구}}) 대비 부족한 산출 데이터·선결 조건을 표로
4. 재반복 필요 유무: 이미 한 것들 중 다시 해야 할 것과 불필요한 것을 근거와 함께 구분
5. 단계 플랜: 우선순위·전제·소요·병렬 가능 여부를 표로 — "지금 당장 가능한 것"과 "협조/선결 필요한 것" 분리
6. 기억에 플랜을 배치 기록해 다음 세션이 물고 가게 하라
```

**실증**: 에피소딕 트랙 갭 분석(C3 증거 3분류 → AB N=6 필요·본런 선결 2건 도출), 집단감독 잔여 판정(S-5 유일 → C-0~C-3 단계표), "재반복 불필요" 판정으로 낭비 런 방지.
