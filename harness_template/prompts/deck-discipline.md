# ④ 발표 덱 규율

**언제 쓰나**: 실험/진행 결과를 발표 슬라이드로 만들 때. (상세 스타일 값은 [`../style/deck-style.md`](../style/deck-style.md) 참조)

## 프롬프트

```
{{주제}} 덱을 다음 규율로 작성/개정하라.
1. 의미 용어 — 코드 내부 식별자 대신 청중이 이해하는 의미 용어 사용
   <!-- 예: 내부명 "memory ladder" → 발표명 "기억 분해 실험(ablation)" -->
   (코드 식별자·과거 기록은 개명하지 않고 보존)
2. 전 슬라이드 "왜/결과" 2줄 구조 — 이 실험을 왜 했나 1줄 + 무엇이 나왔나 1줄
3. 결과 슬라이드에 해석박스 3단: 읽는법 → 보이는 것 → 의미
4. 수치 정직성 — 모든 수치는 {{수치 정본 소스}}와 전수 대조
   <!-- 예: 런별 metrics.json / 실험 registry ★확정 행 -->
   표본 작으면 descriptive 명시, 미실측은 미실측이라 쓴다
5. 완성 후 시각 QA 패스 적용 (→ visual-qa.md)
```

**실증**: 주간 통합 덱 전면 개정 v2 — 전 슬라이드 왜/결과 2줄+해석박스 3단(읽는법→보이는것→의미)+정본 수치 파일과 전수 대조, 용어 확정(내부명→청중 언어, 코드 식별자는 보존).
