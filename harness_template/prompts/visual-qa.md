# ③ 시각 산출물 QA (신선한 눈)

**언제 쓰나**: 덱·차트·도식·PDF 등 렌더링 결과물을 납품하기 전.

## 프롬프트

```
{{산출물 파일}}을 "처음 보는 사람"의 눈으로 시각 검수하라.
- 반드시 실제 렌더 결과(내보낸 PNG/PDF/스크린샷)를 열어서 본다 —
  생성 코드만 읽고 통과 판정 금지
- 사냥 목록: 요소 겹침 / 텍스트 잘림·고아 줄바꿈 / placeholder·더미 텍스트 잔존 /
  빈 영역 / 라벨-데이터 불일치 / 페이지 넘침
- 발견 → 수정 → 재렌더 → 재검수, 지적 0건이 될 때까지 사이클 반복
- 최종 판정은 별도 QA 패스(신선한 컨텍스트)로 한 번 더
```

**실증**: 주간 덱 QA에서 고아 줄바꿈·공백 수정 1사이클, 다른 덱 QA 2라운드 통과, 31장 통합 덱 이중 QA 통과, 그림 8종 재작도 시 4포크 병렬 생성+QA.
