# ② 적대검증/반박 렌즈

**언제 쓰나**: 설계 문서·구현·주장(claim)을 확정하기 전, 스스로 공격해 살아남는 것만 남길 때.

## 프롬프트

```
{{산출물}}에 대해 적대 검증을 수행하라.
- 렌즈 {{N}}개를 지정해 병렬 공격: <!-- 예: 정합성 렌즈 / 재현성 렌즈, 또는 심사자 공격 ①~⑥ -->
- 각 지적은 "왜 현재 상태가 문제인지 + 고치면 뭐가 나아지는지" 근거 필수
  ("수정해라" 단독 금지)
- 현재 표현/설계가 오히려 강점인 경우를 반드시 고려 — 기각 시 기각 근거 기록
- 판정 5분류: 유지 / 강화 / 수정 / 추가 / 삭제
- 에이전트 지적을 나(오케스트레이터)가 재검증한 뒤에만 반영·보고 —
  과잉 지적 필터링 없이 사용자에게 그대로 전달 금지
```

**실증**: 인터페이스 명세 v0.1(8에이전트: 4도메인 병렬 매핑→합성→적대검증 2렌즈→수정 1건 반영), 실험 런북 8종(20에이전트 적대검증 통과), 신규 기능 구현에서 블로커 2건(유령 override·이중 승격) 사전 수정.
