# 세션 재시작·종료 상기 프로토콜

> 장기 멀티세션 프로젝트의 최대 피로원 = 세션 간 맥락 유실. 재시작 시 읽는 순서와 종료 시 남기는 것을 고정한다.

## 재시작 읽기 순서 (세션 시작 직후)

1. **자기 유형 확인**: `CLAUDE.md` §0 세션 표에서 내가 어느 세션인지 → 그 행의 진입점을 따른다
2. **세션 메모리**: `project_session_*` 최신 파일 — 지난 세션이 남긴 "완료한 것 + 다음 할 것"
3. **implementation-notes.md** — 미결 `[빈칸][가정]`과 최근 `[결정][이탈]` (열려 있는 것부터)
4. **EXPERIMENTS.md 진행판·간트** — 트랙 현황과 블로커
5. **관련 정본** (필요 시): 작업 대상의 `specs/` 설계 정본, 환경 문제면 `setups/`

## 기억 검증 규율

- **기억 = 시점 관찰, 라이브 상태 아님** — 코드 동작·파일:라인 인용은 기록 시점의 것. 사실로 단정하기 전에 **현재 코드 실물로 재검증**한다
- 수치는 기억이 아니라 {{registry}} ★확정 행에서 인용
- 기억과 실물이 어긋나면: 실물이 이긴다 → 틀린 기억을 즉시 정정·삭제 (memory-protocol.md 원칙 3)

## 세션 종료 배치 기록 체크리스트

- [ ] 이번 세션의 사실①·결정②·마일스톤④ 배치 기록 (memory-protocol.md 트리거표)
- [ ] `project_session_*` 갱신 또는 신규 — "완료한 것 + 다음 재개 지점" 1~2줄
- [ ] `MEMORY.md` 인덱스 동기화 (신규/변경 포인터, <17KB)
- [ ] `implementation-notes.md` 해결 항목 → 아카이브 1줄 이동, 서사는 정본 문서로 승격
- [ ] 트랙 상태를 바꿨으면 `EXPERIMENTS.md` 진행판·간트 갱신 (바꾼 세션이 책임)
- [ ] 자기 감사 1회: 사전점검 블록 없이 착수한 대형 작업이 있었나 → 있으면 `[이탈]` 기록 (FABLE5.md)
- [ ] 정리 패스: 파편 병합·거짓 기록 삭제·끊긴 포인터 점검 → `🧹 정리:` 1줄
