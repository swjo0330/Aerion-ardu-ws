# 자율 기억/문서 동기화 프로토콜 (AKC)

**목표**: 유저가 "기억해/문서화해"라고 말하지 않아도, 작업 중 발견되는 룰·맥락·결정·함정을 **알아서 판단해** 기록·동기화한다. 유저의 인지 부하 0을 지향.

> **왜 이 구조인가 (실증)**: 실시간·전 저장소 복제로 운영하면 인덱스 비대(리밋 초과)·동일 사실 4중 복제·세션당 기록 10회+가 실측됐다 → **자연 경계 배치 + 정본 1곳 + 즉시 기록은 위험 항목만**으로 완화한 것이 현행.

## 0. 4-저장소 토폴로지 (어디에 무엇이 사는가)

| 저장소 | 위치 | 로드 시점 | 정본 대상 | 쓰기 단위 |
|--------|------|----------|----------|----------|
| **① CLAUDE.md** | 리포 루트 (git 추적) | 매 세션 자동 | **불변 규칙·금지사항·항시 적용 룰** | `## N.` 섹션 / 불릿 |
| **② 파일 메모리** | `~/.claude/projects/{{프로젝트 슬러그}}/memory/*.md` + `MEMORY.md` 인덱스 | 매 세션 (인덱스 주입) | **유저 선호(feedback)·진행 상태(project)·정체성** | 1파일=1사실 |
| **③ {{구조화 메모리 스토어}}** <!-- 예: memoir MCP. 미사용 시: 이 행 삭제 + 아래 트리거표 ①·③의 기록처 "③"을 ②(`project_*`/`knowledge_*` 파일)로 대체 + CLAUDE.md AKC 절의 "4 저장소 라우팅"도 동일하게 3-저장소로 축약 --> | {{스토어 위치}} | 매 세션 키 목록 주입 | **구조화 기술 사실·결정** (점-구분 키) | 키 1개 |
| **④ Docs** | `{{문서 루트}}` (git 추적) | 필요 시 Read | **서사형 작업기록·설계 스펙·실험 결과·환경 셋업** | 날짜 `.md` 파일 |

**라우팅 1원칙 — "수명과 형태로 고른다"**:
- *항시 강제되어야 하는 한 줄 규칙* → ① (가장 비싼 자리, 정말 불변일 때만)
- *나의 행동을 바꾸는 선호/정체성, 세션 간 상태* → ②
- *재인용할 구조화 기술 사실/결정* → ③ (키로 정밀 회상)
- *서술이 필요한 긴 기록(스펙·실험·세션로그)* → ④, 그리고 ②③에 1줄 포인터

**중복 방지**: 동일 사실을 여러 저장소에 전문 복제하지 않는다 — **긴 서사는 ④, 짧은 회상 훅은 ②(인덱스 1줄) 또는 ③(키 1개)**. 이미 있으면 신규 생성 대신 갱신.

## 0-1. 자율성 경계 + 투명성 계약

자율성은 "되돌릴 수 있고 내부에만 영향"인 쓰기에만 부여한다.

| 행위 | 자율 등급 | 통지 방식 |
|------|----------|----------|
| ②③④ 기록·갱신 | **말없이 자동** (가역적 내부 지식) | 응답 말미 `🧠 기억: <slug> — <훅>` 1줄 |
| ① CLAUDE.md 불변 규칙 신설/변경 | **초안만 제시** — 유저가 반영 | "CLAUDE.md §N 추가 제안 — 붙여넣을 텍스트입니다" |
| 코드/설정 수정, 커밋, 외부 전송 | **자율 아님** (유저 승인) | 해당 안 됨 |

**유저 거부권(veto)**: "그거 지워 / 기억하지 마 / 틀렸어" → 즉시 삭제·정정. 자동 기록은 항상 가역적이어야 한다.

## 실행 원칙 6종

1. **자연 경계에서 기록**: ①②④는 마일스톤·방향 전환·세션 종료에 일괄. 즉시는 ③⑤⑥만
2. **중복 회피**: 신규 생성 전 `MEMORY.md` 인덱스·스토어 키 확인 → 기존 갱신 우선
3. **틀린 기억 삭제**: 정정된 사실은 옛 기록을 직접 수정·삭제 (취소선 보존 금지 — 누적 오염 방지)
4. **인덱스 동기화**: 메모리 파일 추가/변경 시 `MEMORY.md` 1줄 포인터 갱신
5. **확신 없으면 묻지 말고 기록 후 통지**: 사소한 판단은 수행 + 1줄 언급, 영향 큰 것만 확인 요청
6. **조용히**: 기록 행위를 장황히 보고하지 않음 — 말미 1줄이면 충분

## 세션 종료/장기 정리 패스 (누적 부패 방지)

- **트리거**: 긴 세션 마무리, compaction 직후, 유저 "정리해"
- **동작**: ①같은 주제 파편 병합 ②거짓이 된 기록 삭제 ③인덱스↔실파일 정합 점검(끊긴 포인터) ④스토어 키 중복 점검
- **원칙**: 정리도 가역적·말없이 자동. 단 삭제는 `🧹 정리: <slug> 병합/삭제` 1줄 통지
