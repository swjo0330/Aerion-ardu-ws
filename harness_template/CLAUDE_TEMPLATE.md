## 0. 세션 체계 (창별 역할 — 세션 시작 시 자기 유형 확인)

<!-- 단일 세션 운용이면 이 표를 지우고 "단일 세션 — 전 영역" 1줄만 남길 것.
     다중 세션 운용이면 아래 3행 패턴을 프로젝트에 맞게 채울 것. -->

목적별 세션(창)을 분리 운용한다. 세션마다 **작업 영역·메모리·문서·진입점이 다르다**.

| 세션 | 역할·작업 영역 | 진입점 (읽는 순서) | 메모리·문서 | 금지 |
|---|---|---|---|---|
| **{{세션A: 총괄 설계·개발}}** (기본) | 설계·구현·하니스·실험·문서·커밋 등 전 영역 | 세션 메모리(`project_session_*` 최신) → `implementation-notes.md` → `EXPERIMENTS.md` (순서 정본 = [memory/session-restart-protocol.md](memory/session-restart-protocol.md)) | 메모리=`project_*`/`feedback_*`, 문서=`{{문서 루트}}` | **{{독점 산출물}} 직접 수정 금지** — 관련 발견은 handoff에 기록만 |
| **{{세션B: 독점 산출물 세션}}** <!-- 예: 논문/릴리즈 문서 세션 --> | {{산출물}} 수정 권한 **독점** | handoff 문서(`{{handoff 경로}}`) 지정 순서 | 메모리=`{{접두사}}_*`, 수치={{registry}} ★확정 행만 | registry 외 수치 인용 금지. 완료 시 handoff 갱신 책임 |
| **{{세션C: 파생 프로젝트}}** | `{{파생 폴더}}` 하위 — 본류 규칙·registry **비적용**, 자체 규칙 우선 | 해당 폴더 자체 CLAUDE.md | 메모리=자체 접두사, 문서=해당 폴더 안에서만 | **본류↔파생 상호 오염 금지**(문서·메모리 양방향) |

- 세션 간 전달: 구현→산출물 = handoff(기록만) / 연속성 = 세션 메모리 + `EXPERIMENTS.md` 진행판 / 수치 = {{registry}}(등록은 사용자 검토 후)

## 1.5 구현 기록 (implementation-notes.md)

- **⚡발동 규칙: 대형 작업(신규 구현·실험 설계/실행·문서/덱 제작·아키텍처 결정·다파일 수정) 지시를 받으면 [FABLE5.md](FABLE5.md)의 "🧠 사전점검 블록"을 응답 첫 부분에 출력하고 시작한다** — 블록이 안 보이면 프로토콜 미이행. 중형=축약 1~3줄, 소형=생략 가능
- **구현 전 Blind Spot Pass**: 코드 만지기 전에 먼저 답한다 — ①놓치고 있을 것·위험한 가정은? ②먼저 확인할 기존 파일·영향 인터페이스는? ③성급 구현 시 깨질 것은? ④최소 수직 슬라이스는? 위험한 가정은 `[가정]`(근거+위험도)으로 선기록
- **구현 중 기록**: 드러난 빈칸·즉석 결정·설계 이탈·게이트 결과를 `implementation-notes.md`에 1줄씩 — `[빈칸][가정][결정][이탈][테스트]`
- **가역성 규칙**: 되돌릴 수 있는 작은 결정 = 기록 후 진행 / 되돌리기 어렵거나 아키텍처·불가침 컴포넌트·{{핵심 실행 인프라}} 영향 = 기록 후 사용자 질문

## 5. 자율 지식 축적 (AKC — 자율 기억)

<!-- 도메인 규칙 팩이 늘어나면 §3·§4 뒤에 §5, §6…으로 추가하고 AKC를 항상 마지막 절로 재번호할 것. -->

> 유저 지시 없이도 작업 중 발견되는 룰·맥락·결정·함정을 **알아서 판단해 기록**한다. 인지 부하 0 지향. 상세 규칙: [memory/memory-protocol.md](memory/memory-protocol.md) <!-- 채택 시 파일 메모리로 이관 가능 -->.

- **6 트리거** — ①도메인 진실 정정/확정 ②설계 결정 확정 ③함정/근본원인 발견 ④단계/마일스톤 완료 ⑤유저 선호/금지 ⑥환경/IP 변경
- **자연 경계 배치** — 기록은 자연 경계(마일스톤 완료·방향 전환·세션 종료)에 묶어서. 즉시 기록은 ③함정·⑤선호/정정·⑥환경만. 사실당 **정본 1곳 전문 + 나머지 1줄 포인터**(전문 복제 금지). MEMORY.md는 순수 인덱스(항목 1~2줄, <17KB 하드 리밋).
- **4 저장소 라우팅** — ①CLAUDE.md=불변 규칙 / ②파일 메모리=선호·정체성·진행상태 / ③{{구조화 메모리 스토어}}=구조화 기술사실·결정 <!-- 예: memoir MCP --> / ④Docs=서사형 기록·스펙·실험. 긴 서사는 ④, 짧은 회상 훅은 ②③. 구현 시점 기록은 `implementation-notes.md`(§1.5).
- **자율성 경계** — ②③④ 기록은 **말없이 자동**(가역적·내부); CLAUDE.md 규칙 신설/코드 수정은 **먼저 통지/승인**.
- **투명성** — 기록한 턴은 응답 말미 `🧠 기억: <slug> — <훅>` 1줄. 유저 "지워/기억하지마"는 즉시 정정·삭제.
- **중복 방지** — 신규 생성 전 `MEMORY.md` 인덱스·스토어 키 확인 → 기존 갱신 우선. 정정된 거짓 기록은 삭제.
- **세션 종료 시** — [memory/session-restart-protocol.md](memory/session-restart-protocol.md)의 "세션 종료 배치 기록 체크리스트"를 수행한다.
